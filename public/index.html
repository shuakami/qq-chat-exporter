QQ Chat Exporter Pro V3

全新的Go版本QQ聊天记录导出工具，更稳定、更强大、更可靠。支持HTML、JSON、TXT多种格式导出，具备实时保存和断点续传功能。

V3版本相比之前有重大改进，建议先全部看完再开始操作。

V3版本新特性

核心改进
独立程序：不再依赖Devtools，使用命令行程序，更稳定可靠
数据库存储：所有数据保存在本地数据库，支持断点续传
实时保存：每30秒自动保存进度，再也不怕数据丢失
多格式导出：支持HTML、JSON、TXT三种格式
时间范围过滤：可以指定导出特定时间段的聊天记录
会话管理：支持多个聊天会话，方便管理不同的导出任务

使用步骤

1. 下载程序

从GitHub下载最新版本的QQ Chat Exporter Pro V3：

访问 https://github.com/shuakami/qq-chat-exporter/releases/latest
下载适合你操作系统的版本（Windows用户下载 qq-chat-exporter.exe）
2. 设置NapCat

V3版本需要NapCat作为QQ接口，按以下步骤详细配置：

第1步：下载 NapCat
  • 访问 GitHub 发布页面：https://github.com/NapNeko/NapCatQQ/releases/latest
  • 下载适合您系统的版本（通常是Windows的 .zip 文件）
  • 解压到任意文件夹（建议桌面或文档文件夹）
第2步：启动 NapCat
  • 双击运行 NapCatWinBootMain.exe
  • 程序启动后会显示二维码
  • 使用手机 QQ 扫码登录
第3步：打开 QQ 设置

登录成功后，在QQ界面中找到并点击设置按钮：

在QQ界面中找到并点击设置按钮
第4步：打开 NapCat 网页界面和配置 WebSocket

在浏览器中打开管理界面地址：

NapCat网页管理界面

然后点击"网络配置"：

在Napcat设置中准备添加Websocket客户端

在网络配置中新建WebSocket客户端：

WebSocket客户端配置界面

重要配置参数：

  • 启用：开启（打开开关）
  • 名称：QQ Chat Exporter
  • URL：ws://localhost:3032
  • 消息格式：Array
  • 重连间隔：5000（重要！必须设置为5000）
  • Token：（留空）
  • 其他选项：保持默认

配置完成后点击保存。

重要提醒：
  • 必须保持NapCat运行状态，V3版本通过WebSocket与QQ通信
  • WebSocket URL必须是 ws://localhost:3032，这是V3版本的默认连接地址
  • 确保QQ处于登录状态，否则无法采集消息
3. 运行导出程序

V3版本使用非常简单，直接双击即可运行：

将下载的 qq-chat-exporter.exe 放到一个专门的文件夹中
双击运行 qq-chat-exporter.exe 文件
首次运行会进入交互式配置向导，按提示设置NapCat连接
配置完成后，程序会显示操作菜单，选择相应功能即可
提示： 如果需要使用命令行参数（如批处理脚本），也可以在终端中运行 ./qq-chat-exporter.exe [命令]

使用流程

双击运行程序后，你会看到交互式菜单，按照以下流程操作：

主菜单选项
开始导出 - 创建新的导出任务
查看历史 - 管理已有的导出任务和会话
系统配置 - 配置NapCat连接和导出设置
帮助信息 - 查看详细使用说明
导出操作步骤
选择聊天类型（好友/群聊）
输入QQ号或群号
选择导出格式（HTML/JSON/TXT）
设置时间范围（可选）
确认开始导出，程序会自动采集和保存
高级用法（命令行）

如果需要批处理或自动化，也支持命令行参数：

# 导出好友聊天记录
./qq-chat-exporter.exe export friend 123456789 --format html

# 导出群聊记录
./qq-chat-exporter.exe export group 987654321 --format json

# 导出指定时间范围
./qq-chat-exporter.exe export friend 123456789 --start-time "2024-01-01" --end-time "2024-12-31"
复制

常用命令

# 查看所有可用命令
./qq-chat-exporter.exe --help

# 查看历史导出记录
./qq-chat-exporter.exe history

# 查看会话列表
./qq-chat-exporter.exe sessions

# 删除指定会话
./qq-chat-exporter.exe sessions delete <session_id>

# 配置NapCat连接（交互式）
./qq-chat-exporter.exe config
复制

控制选项

  • Ctrl + C: 安全中断导出（会保存当前进度）
  • --max-count N: 限制导出消息数量
  • --format html/json/txt: 指定导出格式
  • --output-dir: 指定输出目录

注意事项和使用技巧

V3版本支持图片、文字、表情、@消息等多种消息类型，导出的HTML文件可以直接在浏览器中查看，体验接近原生QQ。

断点续传功能

V3版本最大的优势是支持断点续传：

  • 程序每30秒自动保存进度到数据库
  • 如果程序意外中断，重新运行相同命令会从中断处继续
  • 使用 Ctrl + C 安全退出，不会丢失已采集的数据
  • 可以随时查看采集进度和历史记录

数据管理

  • 所有数据存储在本地SQLite数据库中
  • 支持多个聊天会话同时管理
  • 可以为不同联系人创建不同的导出任务
  • 支持删除过期的会话和导出任务

⚠️ 安全提醒

🎉 根据最新反馈，腾讯现在通常只会发出警告提示，不再进行封号处理。

如果你的聊天数据主要在手机上，建议先在QQ设置中同步聊天记录到电脑，然后再进行导出。

相比旧版本的优势：V3版本不再需要浏览器操作，避免了JS注入的不稳定性，同时数据持久化存储让断点续传成为可能，大大提升了使用体验。

💡 V3版本使用技巧

  • 批量导出：可以为多个好友/群组分别创建导出会话，程序会自动管理
  • 定时导出：定期运行相同命令，只会导出新增的消息（增量备份）
  • 格式选择：推荐HTML格式用于查看，JSON格式用于数据分析
  • 性能优化：程序使用15条/批的小批量请求，平衡速度和稳定性
  • 错误恢复：如果网络中断，重新运行命令会自动从中断处继续

🔧 故障排除

  • 连接失败：检查NapCat是否正常运行，WebSocket配置是否正确
  • 无法获取消息：确认QQ账号已登录，且目标聊天存在
  • 导出空文件：可能是时间范围设置过窄，或该时间段无消息
  • 程序卡住：按 Ctrl+C 安全退出，检查网络连接后重试
