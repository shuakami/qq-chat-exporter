QQ Chat Exporter Pro V4

欢迎使用全新的 V4 版本！基于 NapCatQQ 的聊天记录导出工具，提供现代化的 Web 界面和强大的导出功能。

V4 版本完全重构，集成到 NapCatQQ 框架中，提供更稳定、更强大的导出体验。

V4 版本核心特性

架构升级
集成设计：完全集成到 NapCatQQ 框架中，无需单独安装
Web 界面：基于 React + NextUI 的现代化管理界面
API 驱动：完整的 RESTful API 和 WebSocket 实时通信
数据持久化：SQLite 数据库存储，支持断点续传和任务恢复
定时任务：cron 表达式支持的灵活定时导出系统
多格式导出：HTML、JSON、TXT 三种格式，满足不同需求

使用步骤

1. 下载并启动 NapCatQQ

V4 版本已完全集成到 NapCatQQ 中，您只需下载包含 QCE 功能的版本即可。

访问 项目 Releases 页面 下载最新版本
解压到任意目录，运行 NapCatWinBootMain.exe (Windows)
程序启动后会显示二维码，使用手机 QQ 扫码登录
登录成功后，程序会在后台运行并启动 Web 服务
2. 访问 NapCat 管理界面

登录成功后，访问 NapCatQQ 的 Web 管理界面：

在浏览器中打开 http://localhost:40653 如果你是服务器的话，请开端口40653，然后用服务器ip:40653访问
使用控制台显示的密钥登录管理界面
在管理界面中找到 "QQ 聊天记录导出" 模块入口
3. 进入 QCE V4 界面

通过 NapCat 管理界面访问 QCE V4 功能：

点击侧边栏的 "QQ 聊天记录导出" 进入 V4 界面
首次进入会自动检测 QQ 连接状态和可用会话
4. 开始导出任务

在 V4 界面中，您可以通过以下功能模块完成导出：

仪表板：查看系统状态、任务统计和 NapCatQQ 连接状态
会话管理：浏览好友和群组列表，选择要导出的聊天对象
导出任务：创建、监控和管理导出任务，支持暂停、恢复和下载
定时导出：设置自动化定时任务，支持 cron 表达式和预设周期
系统设置：配置导出参数、存储路径和系统选项

注意事项和使用技巧

数据安全

  • 所有聊天记录的获取和导出过程都在您的本地计算机上完成，数据不会上传到任何服务器，请放心使用。
  • 导出的文件也保存在您的本地硬盘上。

使用技巧

  • 首次使用：建议先到“会话管理”页面，点击“刷新列表”以确保获取到最新的好友和群组信息。
  • 大批量导出：对于包含大量消息的会话，导出可能需要较长时间。您可以在“导出任务”页面观察实时进度。任务在后台运行，您可以关闭浏览器标签页，但不能关闭 NapCatQQ 程序。
  • 定时备份：对于重要的聊天，强烈建议使用“定时导出”功能设置每日或每周的自动备份，以防万一。
  • 格式选择：推荐使用 HTML 格式，因为它能最好地还原聊天界面，方便阅读。JSON 格式适合需要进行二次开发或数据分析的用户。

⚠️ 安全提醒

使用本工具可能违反 QQ 用户协议。根据最新反馈，腾讯现在通常只会发出警告提示，不再进行封号处理，但政策可能随时变化，请自行承担风险。

🔧 故障排除

  • 无法访问 Web 界面：请检查 NapCatQQ 程序是否已成功启动并登录。
  • 无法获取好友/群组列表：请确保您的 QQ 账号在线，并在 Web 界面中尝试点击“刷新列表”。
  • 任务失败：任务失败通常会有错误提示。请检查 NapCatQQ 的日志或 Web 界面上的错误信息以定位问题。
